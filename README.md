# job-hunt-kit

A Claude Code–powered job application system. Search LinkedIn for job posts, generate tailored ATS-optimized resumes, write application emails, and send them — all from your terminal using Claude skills.

---

## What's included

| Skill | Command | What it does |
|---|---|---|
| Setup | `/setup` | One-time setup — collects your profile and generates all config files |
| LinkedIn Scanner | `/scan-linkedin` | Browses LinkedIn, evaluates posts against your requirements, saves qualified ones |
| Resume Generator | `/generate-resume` | Generates a tailored ATS-optimized `.docx` resume from a job description |
| Apply | `/apply` | End-to-end: generate resume + write email + send it |
| Send Email | `/send-email` | Send an application email via Gmail with resume attached |

---

## Requirements

- [Claude Code](https://claude.ai/code) (CLI or VS Code extension)
- Python 3 + `python-docx` (`pip install python-docx`)
- Node.js (for Playwright MCP — used by `/scan-linkedin`)
- A LinkedIn account
- A Gmail account (for `/send-email` and `/apply`)

## Getting started

### 1. Clone the repo

```bash
git clone https://github.com/your-username/job-hunt-kit.git
cd job-hunt-kit
```

### 2. Set up your environment

```bash
cp .env.example .env
```

Edit `.env` and fill in your LinkedIn credentials and search queries:

```
LI_EMAIL=you@example.com
LI_PASSWORD=yourpassword
SEARCH_QUERIES=java developer hiring,senior backend engineer,spring boot opportunity
DAYS_BACK=1
MAX_POSTS_PER_QUERY=20
```

### 3. Open in Claude Code

```bash
claude .
```

### 4. Run setup

```
/setup
```

This collects your work history, contact info, and Gmail credentials. It generates:
- `personal-docs/master_profile.md` — your full experience doc (source of truth for all resumes)
- `personal-docs/user.json` — contact info and config (gitignored)
- `tools/content.py` — your resume bullet library
- `personal-docs/job_tracker.md` — application tracker

### 5. Configure your job requirements

Edit `personal-docs/job_requirements.md` with your target role, tech stack, experience range, location preferences, and work authorization. This file drives `/scan-linkedin`'s evaluation logic.

## Daily workflow

### Find job posts on LinkedIn

```
/scan-linkedin
```

Browses LinkedIn search results using Playwright, evaluates each post against your `job_requirements.md`, and appends qualified ones to `jobs.txt`. Runs in a real browser window — you can watch it work.

Pass a custom query:
```
/scan-linkedin senior java engineer remote
```

### Generate a tailored resume

Drop a job description file into `clients/`, then:
```
/generate-resume
```

Or paste the JD directly and ask Claude to generate a resume from it.

Output lands in `output/applied/[Company] - [Role] - [Date]/`.

### Send an application

```
/apply recruiter@company.com
```

Generates the resume, writes a tailored email, and sends it via Gmail in one shot. Or use `/send-email` if you already have the resume ready.

## File structure

```
.
├── .env                        ← your credentials (gitignored)
├── .env.example                ← template — copy this to .env
├── .mcp.json                   ← Playwright MCP config (pre-configured)
├── jobs.txt                    ← qualified posts saved by /scan-linkedin
│
├── personal-docs/
│   ├── job_requirements.md     ← matching rules for /scan-linkedin (edit this)
│   ├── master_profile.md       ← generated by /setup
│   ├── job_tracker.md          ← generated by /setup
│   └── user.json               ← generated by /setup (gitignored)
│
├── clients/                    ← drop job descriptions here
│
├── output/applied/             ← generated resumes (gitignored)
│
└── tools/
    ├── template.py             ← resume scaffold (copy per application)
    ├── content.py              ← bullet library (generated by /setup)
    ├── runner.py               ← generation entry point
    └── docx_builder.py         ← .docx builder
```

## How the LinkedIn scanner works

`/scan-linkedin` uses the [Playwright MCP](https://github.com/microsoft/playwright-mcp) server to browse LinkedIn in a real browser. Claude drives the browser, extracts post content, evaluates each post against your requirements, and saves qualified ones — no scraping API needed.

The evaluation logic lives in `personal-docs/job_requirements.md`. Edit that file to tune hard blockers, fit signals, and experience requirements for your situation.

---

## Privacy

- `personal-docs/user.json`, `.env`, `output/applied/`, and `clients/applied/` are all gitignored.
- Your master profile, resume files, and credentials never leave your machine unless you push them yourself.
- `jobs.txt` is not gitignored by default — if your saved posts contain sensitive info (recruiter contact details, salary info), add it to `.gitignore`.
